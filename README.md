# claude-cache-warm

Claude Code 智能缓存保活插件。在用户空闲时定期发送轻量请求，保持 prompt cache 热度，避免昂贵的 cache miss。

## 为什么需要这个？

Anthropic prompt cache TTL 为 5 分钟。大上下文（100K-1M tokens）下短暂离开，回来触发 cache miss，成本是 cache hit 的 **22.5 倍**。

| 上下文大小 | 1 次 miss 成本 | 1 次保活成本 | 净省 |
|-----------|--------------|------------|------|
| 100K | $3.38 | $0.15 | $3.23 |
| 200K | $6.75 | $0.30 | $6.45 |
| 500K | $16.88 | $0.75 | $16.13 |

## 安装

在 Claude Code 会话中输入：

```
/plugin install xinnyu/claude-cache-warm
```

### 添加提醒指令（可选）

将以下内容添加到 `~/.claude/CLAUDE.md`，让模型在合适时机自动提醒你启动保活：

```markdown
## 缓存保活提醒
当你注意到以下情况时，提醒用户输入 `/loop /cache-warm:cache-warm` 启动缓存保活：
- 对话已超过 20 轮
- 已读取多个大文件
- 用户可能要短暂离开（如说"等一下"、"我去"、"brb" 等）
提醒一次即可，不要重复提醒。
```

## 使用

在 Claude Code 会话中输入：

```
/loop /cache-warm:cache-warm
```

插件会自动：
- 每 **270 秒**（4.5 分钟）发送一次保活请求，卡在 5 分钟 TTL 内
- 你发消息时自动重置计数（你的消息本身会刷新缓存）
- **20 次后自动停止**（约 90 分钟），超过就不划算了

运行时输出示例：

```
保活 #3/20 | 已运行 ~14min | 预估已省 ~$96.75/M tokens
```

## 工作原理

```
miss 成本 = 未缓存读取($15) + 缓存重建($18.75) = $33.75/M tokens
hit 成本  = $1.50/M tokens
比率      = 22.5x
```

盈亏平衡点：22.5 次保活 = 1 次 miss。使用 20 次阈值留余量，约 90 分钟自动关停。
